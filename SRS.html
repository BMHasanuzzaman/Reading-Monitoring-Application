1. Introduction

 1.1 Purpose

  The purpose of product is to track how long a student reads a certain selection given by the instructor by tracking his
  eyes with AI.This srs describes the whole system of our software. We can know the proper duration of reading
  period. It will be very useful if someone sort those learning materials based on levels of learning.

  The teacher and the user both use this project.This srs is intended for teachers who will evaluate this project. The

 1.4 Product Scope

  This is the product which will used by teachers to check whether a student assigned to read a specific pdf or not and
  how long is the student spending in specific paragraphs on that pdf. It is a tiny, flexible, hard-working tool to
  automate processing & organizing files and folders. There are a group of different files and folders on this
  project. It will give the teachers about information and feedback with data of specific students. We can also
  organize our associations in profiles and automatically scan monitored folders at a defined time interval. We can
  use this product in other sectors.

2. Overall Description

 2.1 Product Perspective

  This is a totally new self constructed product. No other previous version was launched before.

 2.2 Product Functions

  •  Teachers can create an account and open a class with a unique reference code.
  •  Students can create an account.
  •  Students can enroll under a teacher with the required code for a specific class of that teacher.
  •  Students and teachers can sign in using their facebook, email, or phone number.
  •  Teachers can upload pdf files to their specific classes.
  •  Students can open and view the pdf uploaded by their teachers.
  •  Teachers can check which student has spent how much time in reading a specific paragraph of
     a specific pdf.
  •  Snaps will be taken of the students in every 30 seconds so that teachers can monitor if the
     reader is the real one.

 2.3 User Classes and Characteristics

  It is assumed that this product will be mainly used by students and teachers. However it might also
  be used by data analysts of various companies. Students and teachers are more important user
  classes compared to the other users. There will be other classes like sign in, sign up etc.

 2.4 Operating Environment

  This system will operate on android smart phones and on desktops. The android version of the smart
  phones must be at least Kitkat 4.4.4 or more.

 2.5 Design and Implementation Constraints

  In order to effectively use the web version of this system, the user's pc must have a built in webcam or
  a webcam connected to their pc. Developers must have access to firebase since firebase is used for user
  authentication. For developing android versions developers must know java and for the web version
  a php framework. For communication protocol developers must be familiar with git and Bitbucket.
  For using google and facebook API, developers need to be flexiable to that.

 2.6 User Documentation

  A manual in the form of a use case diagram shall be provided with the software.

 2.7 Assumptions and Dependencies

  The API used for image processing and signing in by google in this software will be reused from one of the Google's API.

5. Other Nonfunctional Requirements

 5.1 Performance Requirements

 Mandatory: e.g. response time a valve to close

  • The system is unusable
  • Not mandatory: e.g. response time for a UI
  • The system is usable but provides a non-optimal experience
  • Searching for previous pdf need to be quicker
  • Detecting the movement of face and ensure that it is not the still picture
  • The time duration of reading will be calculated as faster
  • The face cognizance API need to be faster and it will be able to detect the face and eyes movement
    from different angles.
  • It will turn the pages and slides smoothly

 

 5.2 Safety Requirements

 The user need to be behold at the screen when it to work, when the user use the system for a long time it will
 damage the eye sight of the user.To alleviate the risk of damaging eyes the user can use glasses or the brightness
 and the color content will be perfectly optimized according to maintaining the best comfort spectrum turn for eyes.

 5.3 Security Requirements:

 The faster and sooner in the software development process it has to fix the security issues. Application security
 tools that integrate into the application development environment can make this process and workflow simpler and
 more effective. These tools are also useful for compliance audits, since it can save time and the expense by catching
 problems before the auditors seen them.  The user will be notified prior to using the system that this is the sole
 purpose of taking the picture and the picture will not be misused in any other way.

  • The pdf screening quality need to good and acceptable for the user to read.
  • There must be the backup of uploaded files on the system
  • Performance level of the a student will only monitored by the teacher
  • Enrollment of a student should must be ensure by the teacher
  • Each student should have the privacy of informations

 5.4 Software Quality Attributes

 Well documentation for readability. The software will be classify into small part in development stage for further adding
 feature. Upload pdf file and reading file are separate feature from each other for easy testing. It is based on
 multi-platform which increase its availability. Rest API make sure its database flexibility. It is really a effective and
 secure software for educational purpose. In future it can be also used for survey of product for marketing company.

 

 5.5 Business Rules

  • In Every failure case for login/upload file/classroom accessibility will be observed
  • User can give the feedback for the system
  • Application can an collect relevant data from the user to improve the whole performance of the system
